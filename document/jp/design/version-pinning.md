# Design: Version Pinning (Issue #14)

## Goal

LazyNix で個々のパッケージのバージョンを指定できるようにする。
nix-versions をラップし、バージョン検索と flake.nix への統合を提供する。

## Design Decisions

| # | 項目 | 決定 |
|---|------|------|
| 1 | スコープ | flake.nix 統合を優先 |
| 2 | nix-versions 依存 | `nix run github:vic/nix-versions -- ...` で外部実行 |
| 3 | YAML スキーマ | オブジェクト形式に拡張 (破壊的変更) |
| 4 | flake input 命名 | `nixpkgs--<package>--<version>` |
| 5 | 検索 CLI | `lnix search <pkg> --version "<constraint>"` |
| 6 | version の値 | 完全一致のみ |
| 7 | 解決済みコミット保持 | lazynix.yaml に `resolvedCommit`, `resolvedAttr` を自動追記 |
| 8 | パッケージ配置 | stable / unstable / pinned の3セクション |

## Schema Changes

### Before (v0.2.0)

```yaml
devShell:
  allowUnfree: false
  package:
    stable:
      - python312
      - gcc
    unstable:
      - rust-analyzer
  shellHook:
    - "echo Hello"
```

### After (v1.0.0)

```yaml
devShell:
  allowUnfree: false
  package:
    stable:
      - name: python312
      - name: gcc
    unstable:
      - name: rust-analyzer
    pinned:
      - name: go
        version: "1.21.13"
        resolvedCommit: "e607cb5"    # auto-filled by lnix
        resolvedAttr: "go_1_21"      # auto-filled by lnix
  shellHook:
    - "echo Hello"
```

### Rust 型定義

```rust
#[derive(Debug, Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct Package {
    #[serde(default)]
    pub stable: Vec<PackageEntry>,

    #[serde(default)]
    pub unstable: Vec<PackageEntry>,

    #[serde(default)]
    pub pinned: Vec<PinnedPackageEntry>,
}

#[derive(Debug, Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct PackageEntry {
    pub name: String,
}

#[derive(Debug, Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
pub struct PinnedPackageEntry {
    pub name: String,
    pub version: String,

    /// nixpkgs commit hash. Auto-resolved via nix-versions.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub resolved_commit: Option<String>,

    /// Nix attribute path (e.g., "go_1_21"). Auto-resolved via nix-versions.
    #[serde(default, skip_serializing_if = "Option::is_none")]
    pub resolved_attr: Option<String>,
}
```

## Flake Generation

### Generated flake.nix (example)

```nix
# Generated by LazyNix - DO NOT EDIT MANUALLY
{
  description = "DevShell generated by LazyNix";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-25.11";
    nixpkgs-unstable.url = "github:NixOS/nixpkgs/nixos-unstable";
    nixpkgs--go--1-21-13.url = "github:NixOS/nixpkgs/e607cb5";
    flake-utils.url = "github:numtide/flake-utils";
  };

  outputs = { self, nixpkgs, nixpkgs-unstable, nixpkgs--go--1-21-13, flake-utils }:
    flake-utils.lib.eachDefaultSystem (system:
      let
        stablePackages = import nixpkgs {
          inherit system;
          config.allowUnfree = false;
        };
        unstablePackages = import nixpkgs-unstable {
          inherit system;
          config.allowUnfree = false;
        };
        pinnedPkgs-go-1-21-13 = import nixpkgs--go--1-21-13 {
          inherit system;
          config.allowUnfree = false;
        };
      in
      {
        devShells.default = stablePackages.mkShell {
          buildInputs = [
            # Stable packages
            stablePackages.python312
            stablePackages.gcc
            # Unstable packages
            unstablePackages.rust-analyzer
            # Pinned packages
            pinnedPkgs-go-1-21-13.go_1_21
          ];

          shellHook = ''
            echo Hello
          '';
        };
      }
    );
}
```

### Input 命名規則

| 要素 | 規則 | 例 |
|------|------|---|
| flake input name | `nixpkgs--<name>--<version-with-dots-to-hyphens>` | `nixpkgs--go--1-21-13` |
| let binding name | `pinnedPkgs-<name>-<version-with-dots-to-hyphens>` | `pinnedPkgs-go-1-21-13` |
| buildInputs 参照 | `<let-binding>.<resolvedAttr>` | `pinnedPkgs-go-1-21-13.go_1_21` |

バージョン文字列の正規化: `.` を `-` に置換。

## Version Resolution Flow

### `lnix develop` 実行時

```
1. lazynix.yaml を読み込む
2. pinned パッケージを走査
3. resolvedCommit が未設定のエントリを検出
4. nix-versions を呼び出して解決:
   $ nix run github:vic/nix-versions -- --json --one '<name>@<version>'
   → { "installable": "nixpkgs/<commit>#<attr>" }
5. resolvedCommit と resolvedAttr を lazynix.yaml に書き戻す
6. flake.nix を生成 (pinned inputs を含む)
7. nix develop を実行
```

### nix-versions 呼び出し仕様

```bash
nix run github:vic/nix-versions -- --json --one '<name>@<version>'
```

レスポンス例:
```json
[
  {
    "spec": "go@1.21.13",
    "name": "go",
    "version": "1.21.13",
    "installable": "nixpkgs/5ed6275#go_1_21"
  }
]
```

`installable` フィールドから:
- `nixpkgs/<commit>` → `resolvedCommit = "5ed6275"`
- `#<attr>` → `resolvedAttr = "go_1_21"`

### エラーケース

| 状況 | 挙動 |
|------|------|
| 完全一致バージョンが見つからない | エラー終了。利用可能なバージョンの確認に `lnix search` を案内 |
| nix-versions が利用不可 | エラー終了。nix-versions のインストール状況を案内 |
| resolvedCommit が既に存在 | 再解決をスキップ (キャッシュとして利用) |

## CLI Changes

### 新規サブコマンド: `search`

```
lnix search <package-name> [--version "<constraint>"]
```

| フラグ | 説明 | デフォルト |
|--------|------|-----------|
| `--version` / `-v` | セマンティックバージョン制約 | なし (全バージョン表示) |
| `--json` / `-j` | JSON 形式で出力 | false (テーブル出力) |
| `--one` / `-1` | 最新の一致のみ表示 | false |

例:
```bash
# 全バージョンを表示
$ lnix search go

# 制約付き検索
$ lnix search go --version ">=1.20,<1.22"

# JSON で最新の一致のみ
$ lnix search go --version ">=1.20" --json --one
```

内部実装: nix-versions に引数を委譲。

### 既存コマンドへの影響

- `lnix develop`: pinned パッケージの解決ステップを追加
- `lnix init`: テンプレートを新スキーマに更新
- `lnix lint`: pinned パッケージのバリデーションを追加

## Crate-Level Changes

### `flake-generator`

- `config.rs`: `Package`, `PackageEntry`, `PinnedPackageEntry` 型の変更
- `generator.rs`: pinned inputs の生成ロジック追加
- `config.rs` (validate): pinned パッケージのバリデーション追加
  - `name` が有効なパッケージ名であること
  - `version` が空でないこと
  - `resolvedCommit` と `resolvedAttr` はバリデーション対象外 (自動解決のため)

### `cli`

- `cli_parser.rs`: `Search` サブコマンド追加
- `main.rs`: search コマンドのハンドリング追加
- `main.rs`: develop コマンドに version resolution ステップ追加
- lazynix.yaml への書き戻しロジック追加 (resolvedCommit, resolvedAttr)

### `nix-dispatcher`

- nix-versions 呼び出し用の関数追加
- JSON レスポンスのパース型追加

### `linter`

- pinned パッケージの lint 対応 (nix eval で存在確認)

## Migration

v0.2.0 → v1.0.0 は破壊的変更。

- 旧形式の lazynix.yaml は `lnix develop` 実行時にパースエラーとなる
- エラーメッセージで新スキーマへの移行方法を案内する
- `lnix migrate` コマンドの追加は将来の検討事項とし、本 Issue のスコープ外とする
